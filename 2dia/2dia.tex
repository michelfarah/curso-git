\subsection{Principais Características}
\begin{frame}
\frametitle{Principais Características}
\begin{itemize}
  \item <1-> \textbf{Suporte para desenvolvimentos não lineares.}

     Com rápida criação de branches e merges eficientes
 
  \item <2->\textbf{Desenvolvimento distribuído.}
  \item <3->\textbf{Eficiência na manipulação de projetos extensos.}
  \item <4>\textbf{Autenticação criptografada do histórico.}
  
  O histórico é armazenado de tal forma que o nome de uma revisão ('commit'
  para o Git) depende do histórico completo. Uma vez publicado não é possível
  alterar as versões antigas sem que se note esta versão.
\end{itemize}
\end{frame}

\section{Configurando o Git}
\subsection{Primeira vez}
\begin{frame}
\frametitle{Primeira vez}
No Windows:
Abrir o programa \textbf{Git Bash}
\begin{figure}
\centering
\includegraphics[width=4cm]{images/win_passo12.jpg}
\end{figure}

No Linux: Abrir o \textbf{Console}
\end{frame}

\begin{frame}
\frametitle{Primeira vez}
Configurar o nome do usuário:

\fbox{ \textit {\$ git config --global user.name "Nome do usuário"}}

\fbox{ \textit {\$ git config --global user.email email@exemplo.com}}


Colocar cores:

\fbox{ \textit {\$ git config --global color.diff auto}}

\fbox{ \textit {\$ git config --global color.status auto}}

\fbox{ \textit {\$ git config --global color.branch auto}}

Habilitar git-rerere ('reuse recorded resolution') - \textbf{opcional}

\fbox{ \textit {\$ git config --global rerere.enabled 1}}
\end{frame}
