\section{Controlando as versões}

\subsection{Para que serve o controle de versão?}

\begin{frame}
\frametitle{O que é um controlador de versão?}
%\framesubtitle{Subtitles are optional}
- É um software com finalidade de gerenciar diferentes versões no
desenvolvimento de um documento qualquer.

- Muito utilizado por empresas de desenvolvimento de softwares e é util em
diversos aspectos, tanto para projetos pequenos e pessoais como também para
grandes projetos comerciais.

- Os mais comuns são:
\begin{itemize}
  \item <1-> Git;
  \item <3-> SVN;
  \item <4-> SourceSafe (da Microsoft);
  \item <5-> ClearCase (da IBM);
  \item <6-> Entre muitos outros.
\end{itemize}

\end{frame}

\subsection{Como funciona?}

\begin{frame}
\frametitle{Como funciona?}

\begin{figure}[!htb]
     \centering
     \includegraphics[width=5 cm]{1dia/area_trabalho_repositorio.png}
     \caption{Estrutura de um controle de versão é composta por repositório e
     área de trabalho. A comunicação entre elas é através de operações como
     \textit{commit} e \textit{update}}
     \label{Label de referência para a imagem}
\end{figure}

\end{frame}

\begin{frame}
\frametitle{Como funciona?}
- Cada \textit{commit} gera uma \alert{revisão} no repositório, contendo as
modificações feitas, data e autor.

- Cada \alert{revisão} gera uma ``foto'' dos arquivos e diretórios daquele
\textit{commit}, gerando o histórico do projeto. Estas ``fotos'' podem ser
recuperadas e analisadas sempre que desejado.

\end{frame}
